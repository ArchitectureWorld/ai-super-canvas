# Feature 01：图谱式 / 有机式会话画布

## 1. 功能名称

**图谱式 / 有机式会话画布**

英文暂定：

**Organic Graph Chat Canvas**

## 2. 功能定位

这是 AI 超级画板的第一项核心功能，也是后续所有能力的空间底座。

它的目标是：

> 将传统线性 Chat 转化为可视化、可分枝、可回流、可代谢、可重构的有机图谱式会话空间。

## 3. 用户问题

当前 AI Chat 的主要问题是过于线性。

真实使用过程中，用户往往会从一个点出发：

1. 初期沿线性方向推进。
2. 中途逐渐产生分支。
3. 分支继续产生更细的探索。
4. 局部探索形成结论、素材、失败经验或新任务。
5. 这些结果反过来改变主线问题的理解。

传统 Chat 很难表达这种结构。

## 4. 设计目标

### 4.1 必须支持

- 主线对话。
- 从任意语义对象生成分支。
- 分支独立上下文。
- 分支与原语义锚点保持引用关系。
- 分支结果回流主线。
- 低价值分支代谢。
- 主线根据回流内容重构。

### 4.2 必须避免

- 只支持整条消息级别分支。
- 把分支理解为简单复制对话。
- 把删除分支理解为彻底消失。
- 把图谱式会话做成静态思维导图。
- 第一版过早引入复杂多人协作或 Agent 自动调度。

## 5. 核心对象

### 5.1 主线 Mainline

当前项目、问题或任务的主要推进路径。

### 5.2 语义锚点 Semantic Anchor

用户选中的任意可理解对象，包括：

- 词汇
- 短语
- 句子
- 段落
- 图片
- 图片局部区域
- 文件片段
- 画布节点
- 节点关系
- 结论卡
- 任务卡
- 素材卡

### 5.3 分支 Branch

由语义锚点生成的独立探索空间。

### 5.4 卡片 Card

分支探索后的结构化产物。

卡片类型包括：

- 结论卡
- 规则卡
- 任务卡
- 素材卡
- 失败经验卡
- 问题卡
- 决策卡

### 5.5 回流 Feedback / Reintegration

将分支成果反向作用于主线或其它相关节点。

### 5.6 代谢 Metabolism

对低价值、过期、重复或偏离方向的分支进行处理。

代谢方式包括：

- 剪枝
- 落叶
- 休眠
- 腐殖化
- 归档

## 6. MVP 用户流程

### 6.1 创建主线

用户输入一个问题或目标，系统生成主线会话。

### 6.2 选择语义对象

用户在对话中选中一个词汇、句子或段落。

### 6.3 创建分支

用户点击“生成分支”。

系统创建一个分支，并记录：

- 原始语义锚点。
- 原主线位置。
- 分支标题。
- 分支上下文。

### 6.4 分支探索

用户在分支中继续与 AI 讨论。

### 6.5 生成卡片

分支探索到一定阶段后，用户或 AI 将分支结果整理为卡片。

### 6.6 回流主线

用户确认卡片有效后，将其回流到主线。

系统更新：

- 主线摘要。
- 当前问题理解。
- 相关节点关系。
- 后续任务建议。

### 6.7 分支代谢

对于不再需要的分支，用户可选择：

- 剪枝：删除该方向。
- 落叶：标记为自然失效。
- 休眠：暂不推进，未来可恢复。
- 腐殖化：删除完整分支，但提炼有价值养分。
- 归档：完整保留但不参与当前主线。

## 7. UI 原型要求

第一版界面可以包含四个区域：

| 区域 | 作用 |
|---|---|
| 主线区 | 展示当前主线对话 |
| 分支区 | 展示当前激活分支 |
| 画布区 | 展示节点、边、分支关系 |
| 卡片池 | 展示结论卡、任务卡、规则卡、素材卡 |

## 8. 关键交互

### 8.1 选中即锚定

用户选中文本、图片区域或节点时，系统应显示锚点操作菜单。

操作包括：

- 解释
- 扩展
- 反驳
- 生成分支
- 形成任务
- 形成结论
- 关联已有节点

### 8.2 分支独立但不断链

分支应拥有独立上下文，但始终能追溯到原锚点。

### 8.3 回流需要确认

分支结论不应自动污染主线，应先形成卡片，由用户确认后再回流。

### 8.4 代谢保留养分

分支被删除、剪枝或落叶时，系统应提示是否提炼：

- 有效结论
- 失败原因
- 约束条件
- 不再采用的判断
- 可复用素材

## 9. 验收标准

第一版可按以下标准验收：

- 用户可以在一条主线对话中选中任意词汇或句子。
- 用户可以从该锚点创建分支。
- 分支可以独立继续讨论。
- 分支页面能显示来源锚点。
- 分支可以生成结论卡。
- 结论卡可以回流主线。
- 主线能看到回流记录。
- 分支可以被标记为剪枝、落叶、休眠或腐殖化。
- 腐殖化分支可以保留摘要或经验，而不保留完整讨论。

## 10. 暂不解决的问题

- 多用户实时协作。
- 复杂权限系统。
- 完整项目知识库。
- 自动 Agent 调度。
- 图数据库级别复杂推理。
- 复杂动效与视觉隐喻。
- 多模态局部区域精准编辑。

这些问题在后续版本逐步讨论。
