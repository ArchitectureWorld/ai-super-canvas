# 图谱式 / 有机式会话画布

> 文档状态：产品语义背景。Agent-Session 架构生效后，Branch 是带语义锚点的 SessionNode 生长角色；本文件旧的八状态 Branch 生命周期和“落叶等于整个分支”定义已被 [`agent-session-domain-model.md`](./architecture/agent-session-domain-model.md) 取代。

## 1. 双层命名

本功能采用双层命名：

| 层级 | 名称 | 说明 |
|---|---|---|
| 产品机制名 | 图谱式会话 | 强调节点、边、分支、回流、关系网络 |
| 设计哲学名 | 有机式会话 | 强调生长、分枝、落叶、代谢、反哺主干 |

一句话概括：

> 图谱式会话是结构，有机式会话是生命感。

## 2. 为什么不是线性 Chat

传统 Chat 的基本结构是：

```text
用户输入 A
AI 回复 B
用户追问 C
AI 回复 D
用户修正 E
AI 回复 F
```

它默认所有上下文沿同一条时间轴推进。

但真实的 AI 使用过程往往不是线性的，而是从一个问题点出发，先形成主干，再不断生长出侧枝、叶片、果实，并通过局部探索反哺主干。

因此，本功能不是把 Chat 简单做成树形目录，而是把 Chat 升级为：

> 带主干、侧枝、叶片、果实、落叶、腐殖和根系的有机知识网络。

## 3. 不是消息级分支，而是语义对象级分支

分支触发点不应仅限于整条消息。

用户应该可以从任意语义对象创建分支：

| 对象类型 | 示例 | 可能的分支方向 |
|---|---|---|
| 词汇 | 腐殖化、语义锚点、图谱式会话 | 概念解释、命名推敲、机制扩展 |
| 短语 | 有机式会话、知识代谢 | 产品定义、隐喻延展 |
| 句子 | 树叶又会转换能量反哺主干 | 回流机制、能量隐喻、交互方案 |
| 段落 | 一段产品设想 | 拆解、重构、转为 PRD |
| 图片素材 | 参考图、草图、截图 | 风格分支、构图分支、局部修改 |
| 文件片段 | PDF、PPT、MD、PSD 中的一段内容 | 摘要、改写、提取结构、生成任务 |
| 画布节点 | 结论卡、任务卡、素材卡 | 深挖、复用、转文档、回流主线 |
| 局部区域 | 图片中的元素、PPT 页面中的版块 | 局部编辑、替换、图层化、重生成 |

这些对象统称为：

**语义锚点 Semantic Anchor**

## 4. 语义锚点

语义锚点是用户在画布中选中的、可被 AI 理解并作为新分支起点的对象。

语义锚点可以分为：

| 类型 | 说明 |
|---|---|
| 文本锚点 | 词汇、短语、句子、段落 |
| 视觉锚点 | 图片、图像局部区域、设计元素 |
| 文档锚点 | 文件片段、页码区域、表格、标题、引用 |
| 结构锚点 | 画布节点、结论卡、任务卡、素材卡 |
| 关系锚点 | 两个节点之间的边、依赖、反驳、替代、继承关系 |

语义锚点不是静态标注，而是新的生长点。

## 5. 生长流程

图谱式 / 有机式会话的基础流程：

```text
锚定 → 分枝 → 探索 → 回流 → 代谢 → 重构
```

### 5.1 锚定

用户选中一个语义对象，使其成为语义锚点。

### 5.2 分枝

系统以该锚点为起点创建分支。分支拥有独立上下文，但保留与原锚点的引用关系。

### 5.3 探索

用户与 AI 在分支中进行局部讨论、推敲、生成、实验或修改。

### 5.4 回流

分支产生有价值结果后，可以形成结论卡、规则卡、素材卡或任务卡，并反向影响主线。

### 5.5 代谢

低价值、重复、偏离目标或阶段性过期的分支，可以进入剪枝、落叶、休眠或腐殖化流程。

### 5.6 重构

主线根据回流结果重新组织当前问题、目标、判断和下一步行动。

## 6. 历史分支生命周期草案（已被取代）

以下八状态仅保留为早期产品语言研究，不得作为新实现的顶层状态机；当前精确状态以 Agent-Session 领域模型为准。

建议第一版至少支持以下分支状态：

| 状态 | 含义 |
|---|---|
| exploring | 探索中 |
| pending-review | 待确认 |
| integrated | 已回流 |
| dormant | 休眠中 |
| pruned | 已剪枝 |
| decayed | 已落叶 |
| humified | 已腐殖化 |
| archived | 已归档 |

## 7. 产品隐喻：删除不是消失，而是代谢

传统软件中的删除往往意味着内容消失。

在 AI 超级画板中，删除分支应被设计为知识代谢：

| 动作 | 含义 | 植物隐喻 |
|---|---|---|
| 剪枝 | 主动移除不合适方向 | 修剪枝条 |
| 落叶 | 自然淘汰低价值内容 | 叶片代谢 |
| 休眠 | 暂时不推进，但未来可能恢复 | 休眠芽点 |
| 腐殖化 | 不保留完整分支，只提炼养分 | 落叶变腐殖质 |
| 回流 | 将分支结论反哺主线 | 光合作用供能 |
| 结果 | 分支形成稳定成果 | 开花结果 |

最重要的是：

> 分支可以被删除，但其中有价值的养分不应消失。

这里描述的是默认知识代谢体验，不取代受审计的安全删除、保留期限和隐私销毁要求。

## 8. 第一版边界

第一版不追求完整图数据库、不追求复杂自动推理，也不追求多人协作。

第一版只需要打穿以下闭环：

1. 用户可以选中语义对象。
2. 用户可以从语义对象生成分支。
3. 分支可以独立讨论。
4. 分支可以形成结论卡。
5. 结论卡可以回流主线。
6. 无效分支可以代谢。
7. 主线可以根据回流内容更新当前理解。
