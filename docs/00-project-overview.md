# AI 超级画板项目总览

## 1. 项目定位

**AI 超级画板**不是传统绘图软件，也不是普通白板或线性 Chat 工具。它的目标是成为一个面向 AI 原生工作流的**图谱式 / 有机式智能画布**。

它首先解决的问题是：

> 当人与 AI 进行复杂协作时，思考、资料、分支、结论和任务不应该被压缩在一条线性消息流里，而应该在一个可生长、可回流、可代谢的空间中展开。

## 2. 第一核心功能

AI 超级画板的第一个核心功能是：

**图谱式 / 有机式会话画布**

该功能将传统 AI Chat 从线性消息流升级为可视化的语义生长空间。

用户不仅可以从整条消息生成分支，还可以从任意语义对象生成分支，包括：

- 词汇
- 短语
- 句子
- 段落
- 图片素材
- 文件片段
- 画布节点
- 图像局部区域
- 结论卡
- 任务卡
- 规则卡
- 素材卡

这些可被选中、理解和操作的对象统称为：

**语义锚点 Semantic Anchor**

## 3. 总体隐喻

AI 超级画板采用有机生长隐喻：

```text
问题 = 种子
主线 = 主干
词汇 / 句子 / 素材 = 芽点
分支 = 侧枝
局部探索 = 叶片
稳定结论 = 果实
无效分支 = 落叶
失败经验 / 局部判断 = 腐殖质
长期记忆 / 项目知识库 = 根系
回流机制 = 光合作用后的能量输送
```

因此，删除分支不应被理解为简单删除，而应被理解为知识代谢。

低价值分支可以被剪除，但其中有价值的经验、约束、失败判断或局部结论应被提炼出来，转化为养分，反哺主线或沉淀进长期记忆。

## 4. 核心机制

第一功能的核心机制为：

```text
锚定 → 分枝 → 探索 → 回流 → 代谢 → 重构
```

具体含义：

1. 用户围绕一个主问题展开对话。
2. 系统将对话、文本、图片、文件、素材和结论转化为可操作的语义对象。
3. 用户可以从任意词汇、句子、段落、素材、节点或局部区域创建分支。
4. 分支拥有独立上下文，但保留与原始语义锚点的引用关系。
5. 分支探索后可以形成结论卡、素材卡、任务卡或规则卡。
6. 有价值结果可以回流主线，或连接到其它相关节点。
7. 无效分支可以通过剪枝、落叶、休眠或腐殖化进行代谢。
8. 主线根据回流内容重构当前理解。

## 5. 后续功能承载

图谱式 / 有机式会话画布是后续所有能力的空间底座，包括但不限于：

- AI 绘图：方案分支、风格分支、局部修改分支。
- 图像分层：图层、元素、遮罩、局部区域与原图锚点关系。
- PPT 工作流：主题、结构、版式、图像、文字与审核意见回流。
- Rhino 看图建模：效果图、语义图、尺度确认、建模循环与视角比对。
- Agent 协作：Hermes、OpenClaw、Codex 等多 Agent 任务网络。
- 知识沉淀：稳定结论、失败分支、经验规则、长期记忆。

## 6. 当前阶段原则

当前阶段应控制颗粒度，优先打穿第一功能的最小闭环：

- 不急于做完整大而全画布。
- 不急于引入复杂多人协作。
- 不急于做完整知识库。
- 不急于做自动 Agent 调度。
- 不急于做复杂视觉动效。

第一版应优先让一个用户和一个 AI 的对话从线性变成图谱式。
