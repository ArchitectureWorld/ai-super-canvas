# AI 超级画板产品路线图

## 0. 路线图原则

AI 超级画板不应一开始就做成复杂大系统。

当前阶段的重点是：

> 先打穿图谱式 / 有机式会话画布的最小闭环，再逐步承载绘图、PPT、建模、Agent 协作和知识沉淀。

## 1. V0.1：图谱式 / 有机式会话 MVP

### 目标

让一个用户和一个 AI 的对话从线性消息流变成图谱式会话空间。

### 核心能力

- 线性主线对话。
- 对话内容自动形成可选语义对象。
- 支持从词汇、句子、段落生成分支。
- 分支拥有独立上下文。
- 分支保留与原语义锚点的引用关系。
- 分支可形成结论卡。
- 结论卡可回流主线。
- 分支可被剪枝、落叶、休眠或腐殖化。

### 不做

- 不做复杂多人协作。
- 不做完整知识库。
- 不做自动 Agent 调度。
- 不做复杂图数据库推理。
- 不做过度视觉动效。

## 2. V0.2：素材与文件锚点

### 目标

让图片、文件、PPT 页面、PDF 片段也成为可分枝的语义对象。

### 核心能力

- 上传素材形成素材卡。
- 文件内容形成文档锚点。
- 图片局部区域形成视觉锚点。
- 支持从素材卡生成讨论分支。
- 支持从图片局部生成修改分支。
- 支持素材与结论卡互相引用。

## 3. V0.3：结论卡与知识代谢

### 目标

让分支讨论不只是留下聊天记录，而是沉淀为结构化成果。

### 核心能力

- 结论卡 Conclusion Card。
- 规则卡 Rule Card。
- 任务卡 Task Card。
- 素材卡 Asset Card。
- 失败经验卡 Failure Card。
- 分支腐殖化：删除完整分支，但提炼可复用养分。
- 主线重构：根据回流内容更新当前主问题、目标和判断。

## 4. V0.4：AI 绘图工作流接入

### 目标

将图谱式会话画布用于 AI 图像生成与修改流程。

### 核心能力

- 从文本锚点生成图像分支。
- 从图片锚点生成风格分支。
- 从图像局部区域生成局部修改分支。
- 多方案并行比较。
- 生成结果回流到主线设计判断。

## 5. V0.5：PPT / 设计工作流接入

### 目标

承载 AI PPT 和设计排版流程。

### 核心能力

- 主题结构分支。
- 页面版式分支。
- 图片风格分支。
- 文案优化分支。
- 审核意见回流。
- 与图像分层、PPT 可编辑化工作流衔接。

## 6. V0.6：Rhino / 建模工作流接入

### 目标

承载看图建模与循环校核工作流。

### 核心能力

- 效果图 / 实景照片作为视觉锚点。
- 初步语义图生成。
- 尺度、材质、构造等人工确认。
- Rhino / Grasshopper 建模任务分支。
- 特定视角图像比对。
- 建模结果回流主线。

## 7. V0.7：Agent 协作空间

### 目标

将 Hermes、OpenClaw、Codex 等 Agent 协作过程接入图谱式画布。

### 核心能力

- 按项目 / 执行事项形成任务群。
- Hermes 负责目标理解、任务组织和边界控制。
- OpenClaw Agent Team 承接专业执行。
- Codex 承接代码执行。
- Agent 讨论、执行反馈、成果汇报形成节点。
- 任务分支可回流主线。

## 8. V1.0：AI 原生工作画布

### 目标

形成可持续生长的 AI 工作空间。

### 核心能力

- 图谱式会话。
- 多模态语义锚点。
- 素材 / 文件 / 图片 / 代码 / 任务统一挂接。
- 分支探索与结果回流。
- 知识代谢与长期记忆。
- Agent 协作与执行闭环。

## 9. 当前优先级

当前最优先事项：

1. 明确 Feature 01 的交互模型。
2. 明确节点、边、分支、锚点的数据结构。
3. 设计第一版画布 UI。
4. 做一个最小可点击原型。
5. 再决定技术栈和后续开发路线。
