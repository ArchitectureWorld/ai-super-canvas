# 动态生长层

## 1. 为什么需要动态生长层

TapNow 类分析通常更容易提炼出静态结构：

- 节点
- 连线
- 素材库
- 历史面板
- 底部输入区
- 上下文菜单
- Workflow
- 资产库

这些内容解决的是：

```text
画布上有什么
```

但 AI 超级画板还必须回答：

```text
画布如何生长
```

因此，AI 超级画板需要一个明确的 **Dynamic Growth Layer / 动态生长层**。

## 2. 动态生长层定义

动态生长层负责定义：

- 对象如何出现
- 状态如何变化
- 关系如何流动
- AI 如何执行
- 用户如何确认
- 分支如何代谢
- 主线如何重构

简单区分：

```text
静态层 = Node / Edge / Card / Panel / Asset
动态层 = Growth / Flow / State / Feedback / Metabolism / Reconstruction
```

## 3. 生长动态

核心流程：

```text
提出问题
→ 生成主干
→ 选中语义锚点
→ 长出分支
→ 分支继续生长
→ 形成叶片探索
→ 结论结果实
→ 回流主干
→ 主干重构
```

重点不是“新增一个节点”，而是让用户理解：

> 这个分支是从哪个语义点生长出来的。

典型交互：

```text
选中一句话
→ 句子边缘出现微弱高亮
→ 点击“生成分支”
→ 画布中从该句子锚点长出一条轻线
→ 新分支节点展开
→ 分支进入探索状态
```

## 4. 状态动态

分支、节点、卡片都应拥有生命周期。

| 状态 | 动态表现 |
|---|---|
| 探索中 | 节点保持活跃，高亮边线 |
| 待确认 | 卡片进入审核态，等待用户确认 |
| 已回流 | 分支到主线出现回流线 |
| 休眠 | 分支变淡，但保留唤醒入口 |
| 剪枝 | 分支收缩并从主视图移除 |
| 落叶 | 分支逐渐淡出，进入代谢区 |
| 腐殖化 | 分支压缩为经验 / 规则 / 约束卡 |
| 归档 | 保留完整内容，但退出当前主线 |

## 5. AI 执行动态

AI 不只是回复文本，而是在画布中执行一系列可观察动作。

例如：

- AI 正在理解锚点
- AI 正在扩展分支
- AI 正在生成结论卡
- AI 正在检测是否可回流
- AI 正在提炼腐殖化养分
- AI 建议剪枝某个分支
- AI 发现两个分支可以合并

表现方式可以包括：

- 思考中节点
- 执行中边线
- 待确认卡片
- 建议浮层
- 回流预览
- 变更高亮

核心原则：

> AI 的动作必须可见、可中断、可确认、可回滚。

## 6. 用户操作动态

用户的核心动作链：

```text
选中 → 锚定 → 分枝 → 探索 → 结论化 → 回流 / 腐殖化
```

示例：

```text
1. 选中“腐殖化”这个词
2. 出现语义锚点浮层
3. 右侧 Inspector 显示锚点信息
4. 用户点击“生成分支”
5. 画布出现新分支
6. AI 在分支中展开讨论
7. 用户确认生成术语卡
8. 术语卡进入卡片池
9. 用户选择回流主线
10. 主线定义被更新
```

用户不是点一堆按钮，而是在修剪一张正在生长的智能画布。

## 7. 图谱拓扑动态

图谱不是固定的节点图，而应该随工作推进不断改变结构。

需要支持：

- 主线 → 分支
- 分支 → 主线
- 分支 → 分支
- 叶片 → 根系
- 旧结论 → 新任务
- 失败经验 → 约束规则
- 创意执行结果 → 主线判断
- 生成失败 → 腐殖化经验

这体现了一个关键原则：

> 树形结构不仅仅是上下层级，其本质还是网状。

## 8. 时间动态

AI 超级画板至少有三种时间：

| 时间类型 | 含义 |
|---|---|
| 操作时间 | 用户做了什么，可撤销 / 重做 |
| 生长时间 | 分支如何产生、发展、回流、代谢 |
| 知识时间 | 哪些结论稳定了，哪些过期了，哪些进入长期记忆 |

建议后续设计一个：

```text
Growth Timeline / 生长时间线
```

记录：

- 什么时候长出分支
- 什么时候形成结论
- 什么时候回流主线
- 什么时候剪枝
- 什么时候腐殖化
- 什么时候主线重构

## 9. 与统一工作台的关系

动态生长层不是独立页面，而是贯穿统一工作台的底层机制。

在不同焦点模式下，动态生长层表现不同：

| 焦点模式 | 动态重点 |
|---|---|
| 生长 | 分枝、探索、回流、主线重构 |
| 创意 | 节点执行、生成状态、版本分支 |
| 资产 | 资产沉淀、来源追踪、复用关系 |
| 任务 | 任务状态、执行链路、Agent 反馈 |
| 复盘 | 剪枝、落叶、腐殖化、经验沉淀 |
| 历史 | 操作历史、生长时间线、版本恢复 |

## 10. 设计原则

动态生长层应遵守：

1. 任何分支都应可追溯来源锚点。
2. 任何 AI 动作都应可见、可确认、可回滚。
3. 任何回流都应有用户确认或至少可撤销。
4. 任何删除都应优先被理解为代谢，而不是消失。
5. 任何创意执行结果都应能回流主线、进入资产、进入历史或腐殖化为经验。
6. 任何长期稳定结论都应有机会进入根系层。
