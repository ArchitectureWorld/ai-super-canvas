# UI / UX 方向稳定版

## 1. 总体方向

AI 超级画板第一版界面方向稳定为：

> **一个统一工作台，不同功能侧重点。**

更完整的产品表述：

> AI 超级画板不是“上游思考工作台 + 下游创意工作台”的组合，而是一张统一的有机式 AI 工作画布。TapNow 类节点生成、素材管理、历史版本和 Workflow 能力，属于这张统一画布中的“创意执行侧重点”。用户不需要离开当前工作台，而是在同一画布中从思考生长自然过渡到创意执行，再从执行结果回流到主干。

## 2. UI 参考关系

当前参考关系修正为：

```text
主产品 UI：Apple 的高级秩序感 + 有机生长机制
创意执行侧重点：参考 TapNow 的节点式创意画布能力
统一原则：所有能力都发生在同一个工作台里
```

原先“Apple 的高级秩序感 + TapNow 的画布创作感 + 有机生长机制”仍然有参考价值，但需要防止被误读为“把 TapNow 当成产品母体”。

更准确的理解是：

- Apple 提供外层视觉秩序、克制感和高级感。
- 有机式会话提供主产品的核心机制。
- TapNow 提供统一画布中“创意执行侧重点”的节点、连线、资产、历史和 Workflow 参考。

## 3. Apple 方向：高级秩序感

Apple 官网提供的是外层视觉气质与信息组织原则。

可吸收的设计特征：

| Apple 特征 | 转译到 AI 超级画板 |
|---|---|
| 极简顶部导航 | 不堆功能入口，只保留项目、搜索、视图、设置 |
| 大留白 | 画布不要塞满节点，给思考留呼吸感 |
| 强视觉中心 | 当前主线、当前分支、当前任务永远是焦点 |
| 短句表达 | 节点、卡片、按钮文字都要克制 |
| 高级卡片感 | 结论卡、任务卡、素材卡要像产品卡片，不像表格 |
| 轻动效 | 分枝、回流、落叶、生成、版本切换都要轻微、精致、有意义 |

核心吸收：

> Apple 提供“高级、克制、清晰、有秩序”的外层气质。

## 4. TapNow 方向：创意执行侧重点参考

TapNow 提供的是节点式创意画布的执行能力参考，而不是 AI 超级画板的完整产品母体。

可吸收的设计特征：

| TapNow 方向 | 转译到 AI 超级画板 |
|---|---|
| Canvas-first | 创意执行时以画布节点组织输入、生成和结果 |
| 节点化生成 | 文本、图片、生成器、编辑器、版本都可以成为节点 |
| 连线表达依赖 | Prompt、参考图、目标节点之间通过连线表达输入关系 |
| 资产管理 | 素材、历史结果、生成产物可在统一画布中复用 |
| 历史与版本 | 生成结果、失败尝试、版本分支可以被找回和回流 |
| Workflow | 一组节点、连线和参数可以沉淀为可复用流程 |

核心吸收：

> TapNow 提供“创意执行侧”的节点、连线、素材、历史和 Workflow 参考。

## 5. 统一工作台模型

AI 超级画板应被理解为：

```text
统一画布 Workspace
├─ 语义生长侧重点
├─ 创意执行侧重点
├─ 资产记忆侧重点
├─ 任务组织侧重点
└─ 复盘沉淀侧重点
```

这些不是不同产品，也不是互相割裂的页面，而是同一张画布中的不同工作焦点。

### 5.1 语义生长侧重点

突出：

- 主干
- 分支
- 语义锚点
- 结论卡
- 回流线
- 腐殖化记录

适合处理：

- 问题如何展开
- 概念如何生长
- 分支如何探索
- 结论如何稳定
- 主线如何重构

### 5.2 创意执行侧重点

突出：

- 文本节点
- 图片节点
- 生成节点
- 编辑节点
- 参考图节点
- 版本节点
- 模型参数

适合处理：

- 图片生成
- 局部编辑
- 多方案生成
- 版本比较
- 结果资产化

### 5.3 资产记忆侧重点

突出：

- 素材库
- 历史结果
- 项目资产
- 全局资产
- 收藏
- 标签
- 来源关系
- 复用记录

适合处理：

- 跨项目素材复用
- 生成历史找回
- 参考图管理
- 风格、材质、构图、Prompt 沉淀

### 5.4 任务组织侧重点

突出：

- 任务卡
- Workflow
- 执行状态
- Agent 分工
- 待确认事项

适合处理：

- 把讨论成果变成任务
- 把任务组织成执行链路
- 追踪执行状态
- 让 Agent 或工具参与执行

### 5.5 复盘沉淀侧重点

突出：

- 回流记录
- 失败经验
- 剪枝记录
- 腐殖化养分
- 稳定规则
- 长期记忆候选

适合处理：

- 哪些探索有效
- 哪些分支被放弃
- 哪些失败经验应沉淀
- 哪些结论应进入长期记忆

## 6. 第一版界面结构

第一版仍建议采用统一工作台结构：

```text
┌──────────────────────────────────────────────┐
│ Top Bar：项目名 / 搜索 / 焦点模式 / 设置       │
├───────────────┬──────────────────┬───────────┤
│ 主线区         │    统一画布       │ 对象操作区 │
│ Mainline      │    Canvas        │ Inspector │
├───────────────┴──────────────────┴───────────┤
│ 底部卡片池：结论卡 / 任务卡 / 规则卡 / 素材卡   │
└──────────────────────────────────────────────┘
```

其中“焦点模式”不是切换产品，而是调整同一画布中不同对象的视觉权重。

建议初始焦点模式：

```text
生长 | 创意 | 资产 | 任务 | 复盘
```

## 7. 区域定义

### 7.1 Top Bar

顶部导航应极简、低存在感，不抢画布焦点。

建议只保留：

- 项目名
- 当前项目 / 当前工作空间
- 搜索
- 焦点模式切换
- 设置

### 7.2 Canvas

中心统一画布是核心区域。

画布中可呈现：

- 主干节点
- 分支节点
- 语义锚点
- 结论卡
- 任务卡
- 素材卡
- 文本节点
- 图片节点
- 生成节点
- 编辑节点
- 版本节点
- 回流线
- 依赖线
- 代谢状态

视觉方向是：

> 高级工具感 + 有机生长隐喻 + 可执行节点网络。

不能做成游戏树，也不能做成传统流程图软件。

### 7.3 Mainline

左侧主线区不是普通聊天记录，而是主干结构。

它显示：

- 当前主问题
- 关键对话节点
- 已回流结论
- 主线摘要
- 当前判断
- 下一步任务

核心作用：

> 让用户永远知道主干正在往哪里生长。

### 7.4 Inspector

右侧对象操作区用于显示当前选中的语义锚点、节点、卡片或素材。

当用户选中一个词、句子、节点、图片区域时，右侧出现操作面板。

典型操作包括：

- 解释
- 扩展
- 反驳
- 生成分支
- 形成任务
- 形成结论
- 关联已有节点
- 回流主线
- 腐殖化
- 生成 / 编辑
- 保存为资产
- 查看版本

### 7.5 Card Pool

底部卡片池是“果实区”。

用于承载：

- 结论卡
- 规则卡
- 任务卡
- 素材卡
- 失败经验卡
- 决策卡

分支探索不应只留下聊天记录，而应沉淀为卡片，再由用户决定是否回流主线、转为任务、进入创意执行，或腐殖化为经验。

## 8. 第一版视觉关键词

```text
克制
通透
留白
轻阴影
圆角
低噪声
强层级
局部动效
高级卡片
统一画布
焦点切换
```

## 9. 第一版色彩方向

建议第一版色彩保持低饱和、低噪声。

```text
背景：近白 / 浅灰 / 深色模式近黑
主线：深灰 / 黑
分支：低饱和蓝紫
回流：低饱和绿色
创意执行：低饱和蓝 / 紫
代谢：灰绿 / 浅棕
警告：低饱和橙
```

整体应避免过度彩色化，不能像流程图软件，也不能像普通白板。

## 10. 核心 UX 原则

稳定原则：

> **用户不是在管理聊天记录，也不是在切换多个工作台，而是在同一张智能画布里不断生长、执行、修剪和沉淀。**

因此，第一版交互动作应围绕：

```text
选中
锚定
分枝
探索
结论化
转任务
创意执行
回流
剪枝
腐殖化
重构
```

而不是围绕：

```text
新建聊天
复制消息
删除消息
搜索聊天
切换工作台
```

## 11. MVP 界面收敛

第一版不要做复杂无限画布。

优先做清晰的：

```text
左：主线
中：统一画布
右：对象操作
下：卡片池
上：焦点模式
```

第一版优先支持：

- 文本锚点
- 文本分支
- 结论卡
- 任务卡
- 回流主线
- 分支代谢
- 焦点模式雏形

创意执行侧可以先不完整实现 TapNow 级能力，但文档和架构上必须预留其作为同一画布内能力域的位置。
