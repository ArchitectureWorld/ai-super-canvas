# UI / UX 方向稳定版

## 1. 总体方向

AI 超级画板第一版界面方向稳定为：

> **Apple 的高级秩序感 + TapNow 的画布创作感 + AI 超级画板自己的有机生长机制。**

更完整的产品表述：

> AI 超级画板第一版采用 Apple 式高级克制的外层视觉语言，结合 TapNow 式 Canvas-first 的创作交互。整体界面以画布为中心，以语义锚点为操作入口，以分支生长、结论回流和知识代谢为核心 UX。

## 2. Apple 方向：高级秩序感

Apple 官网提供的是外层视觉气质与信息组织原则。

可吸收的设计特征：

| Apple 特征 | 转译到 AI 超级画板 |
|---|---|
| 极简顶部导航 | 不堆功能入口，只保留项目、搜索、视图、设置 |
| 大留白 | 画布不要塞满节点，给思考留呼吸感 |
| 强视觉中心 | 当前主线 / 当前分支永远是焦点 |
| 短句表达 | 节点、卡片、按钮文字都要克制 |
| 高级卡片感 | 结论卡、任务卡、素材卡要像产品卡片，不像表格 |
| 轻动效 | 分枝、回流、落叶都要轻微、精致、有意义 |

核心吸收：

> Apple 提供“高级、克制、清晰、有秩序”的外层气质。

## 3. TapNow 方向：画布创作感

TapNow 提供的是画布优先、即时操作、创作流动的交互感。

可吸收的设计特征：

| TapNow 方向 | 转译到 AI 超级画板 |
|---|---|
| Canvas-first | 主界面首先是画布，不是聊天列表 |
| 即时生成 | 选中词、句子、素材后立即出现操作 |
| 局部操作 | 任意语义对象都可以成为分支起点 |
| Agentic 操作感 | AI 是画布里的执行能力，不只是回复文本 |
| 结果可视化 | 分支、卡片、素材、关系都直接在画布上呈现 |
| 创作流 | 用户不是填表，而是在画布中不断生成、修剪、回流 |

核心吸收：

> TapNow 提供“画布优先、即时操作、创作流动”的交互感。

## 4. 第一版界面结构

第一版建议采用：

```text
┌──────────────────────────────────────────────┐
│ Top Bar：项目名 / 搜索 / 视图模式 / 设置       │
├───────────────┬──────────────────┬───────────┤
│ 主线区         │    图谱画布       │ 对象操作区 │
│ Mainline      │    Canvas        │ Inspector │
├───────────────┴──────────────────┴───────────┤
│ 底部卡片池：结论卡 / 任务卡 / 规则卡 / 素材卡   │
└──────────────────────────────────────────────┘
```

## 5. 区域定义

### 5.1 Top Bar

顶部导航应极简、低存在感，不抢画布焦点。

建议只保留：

- 项目名
- 当前项目 / 当前工作空间
- 搜索
- 视图模式切换
- 设置

### 5.2 Canvas

中心图谱画布是核心区域。

画布中可呈现：

- 主干节点
- 分支节点
- 语义锚点
- 结论卡
- 任务卡
- 素材卡
- 回流线
- 代谢状态

视觉方向是：

> 高级工具感 + 有机生长隐喻。

不能做成游戏树，也不能做成传统流程图软件。

### 5.3 Mainline

左侧主线区不是普通聊天记录，而是主干结构。

它显示：

- 当前主问题
- 关键对话节点
- 已回流结论
- 主线摘要
- 当前判断
- 下一步任务

核心作用：

> 让用户永远知道主干正在往哪里生长。

### 5.4 Inspector

右侧对象操作区用于显示当前选中的语义锚点、节点、卡片或素材。

当用户选中一个词、句子、节点、图片区域时，右侧出现操作面板。

典型操作包括：

- 解释
- 扩展
- 反驳
- 生成分支
- 形成任务
- 形成结论
- 关联已有节点
- 回流主线
- 腐殖化

### 5.5 Card Pool

底部卡片池是“果实区”。

用于承载：

- 结论卡
- 规则卡
- 任务卡
- 素材卡
- 失败经验卡
- 决策卡

分支探索不应只留下聊天记录，而应沉淀为卡片，再由用户决定是否回流主线。

## 6. 第一版视觉关键词

```text
克制
通透
留白
轻阴影
圆角
低噪声
强层级
局部动效
高级卡片
```

## 7. 第一版色彩方向

建议第一版色彩保持低饱和、低噪声。

```text
背景：近白 / 浅灰 / 深色模式近黑
主线：深灰 / 黑
分支：低饱和蓝紫
回流：低饱和绿色
代谢：灰绿 / 浅棕
警告：低饱和橙
```

整体应避免过度彩色化，不能像流程图软件，也不能像普通白板。

## 8. 核心 UX 原则

稳定原则：

> **用户不是在管理聊天记录，而是在修剪一个正在生长的智能画布。**

因此，第一版交互动作应围绕：

```text
选中
锚定
分枝
探索
结论化
回流
剪枝
腐殖化
重构
```

而不是围绕：

```text
新建聊天
复制消息
删除消息
搜索聊天
```

## 9. MVP 界面收敛

第一版不要做复杂无限画布。

优先做清晰的：

```text
左：主线
中：画布
右：对象操作
下：卡片池
```

第一版只支持：

- 文本锚点
- 文本分支
- 结论卡
- 回流主线
- 分支代谢

图片、文件、PPT、Agent 协作后续再扩展。
