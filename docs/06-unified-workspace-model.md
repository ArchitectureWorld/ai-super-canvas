# 统一工作台模型

## 1. 稳定结论

AI 超级画板不是多个工作台的拼接，而是一张统一的有机式 AI 工作画布。

稳定表述：

> **工作台只有一个，但在同一个工作台里存在不同功能侧重点。**

也就是说，AI 超级画板不是：

```text
上游认知工作台
→ 下游 TapNow 式创意工作台
→ 结果再回流
```

而应该是：

```text
一个统一工作台
├─ 思考生长侧重点
├─ 创意执行侧重点
├─ 素材资产侧重点
├─ 任务组织侧重点
├─ 复盘沉淀侧重点
└─ 历史版本侧重点
```

## 2. TapNow 的正确位置

TapNow 类能力不应被理解为另一个“下级工作台”。

更准确的定位是：

> **TapNow 类节点生成、素材管理、历史版本和 Workflow 能力，属于 AI 超级画板统一工作台中的“创意执行侧重点”。**

TapNow 主要参考的是：

- 文本节点
- 图片节点
- 生成节点
- 编辑节点
- 连线依赖
- 资产库
- 历史结果
- Workflow
- 底部 AI 输入

这些能力服务于创意执行，但不等于 AI 超级画板的全部。

## 3. 一张画布，多种工作焦点

AI 超级画板应采用：

```text
同一画布 + 不同焦点模式
```

不是：

```text
进入思考工作台
进入创意工作台
进入资产工作台
```

而是：

```text
在同一张画布中切换显示重点、操作重点和视觉权重。
```

建议的焦点模式：

```text
生长 | 创意 | 资产 | 任务 | 复盘 | 历史
```

焦点模式不是切换产品，而是调整同一画布中不同对象的权重。

## 4. 功能侧重点划分

### 4.1 思考生长侧重点

核心对象：

- 主干
- 分支
- 语义锚点
- 结论卡
- 问题卡
- 回流线
- 腐殖化记录

主要任务：

- 问题如何展开
- 概念如何生长
- 分支如何探索
- 结论如何稳定
- 主线如何重构

### 4.2 创意执行侧重点

核心对象：

- 文本节点
- 图片节点
- 参考图节点
- 生成节点
- 编辑节点
- 版本节点
- 模型参数

主要任务：

- 图片生成
- 局部编辑
- 多方案生成
- 版本比较
- 结果资产化

### 4.3 素材资产侧重点

核心对象：

- 素材库
- 历史结果
- 项目资产
- 全局资产
- 收藏
- 标签
- 来源关系
- 复用记录

主要任务：

- 跨项目素材复用
- 生成历史找回
- 参考图管理
- 风格、材质、构图、Prompt 沉淀

### 4.4 任务组织侧重点

核心对象：

- 任务卡
- Workflow
- 执行状态
- Agent 分工
- 待确认事项

主要任务：

- 把讨论成果变成任务
- 把任务组织成执行链路
- 追踪执行状态
- 让 Agent 或工具参与执行

### 4.5 复盘沉淀侧重点

核心对象：

- 回流记录
- 失败经验
- 剪枝记录
- 腐殖化养分
- 稳定规则
- 长期记忆候选

主要任务：

- 哪些探索有效
- 哪些分支被放弃
- 哪些失败经验应沉淀
- 哪些结论应进入长期记忆

### 4.6 历史版本侧重点

核心对象：

- 操作历史
- 生成历史
- 分支生长历史
- 版本历史
- 主线重构记录

主要任务：

- 撤销 / 重做
- 找回生成结果
- 恢复某个版本
- 复盘生长过程
- 理解关键决策如何形成

## 5. 同一对象在不同焦点中的变化

同一个对象不应该因为焦点切换而复制成多个对象。

例如一个“结论卡”：

| 焦点模式 | 结论卡表现 |
|---|---|
| 生长 | 作为主干或分支的稳定结果 |
| 创意 | 可转化为 Prompt、生成约束或执行输入 |
| 任务 | 可转化为任务卡或 Workflow 步骤 |
| 复盘 | 可查看来源分支、回流时间和影响范围 |
| 历史 | 可查看版本变化和主线重构记录 |

这意味着：

> 焦点模式改变的是对象的显示权重和可操作入口，不改变对象本身的唯一性。

## 6. 统一工作台中的典型流转

一个典型流程可以是：

```text
问题输入
→ 生成主干
→ 选中语义锚点
→ 长出分支
→ 分支形成结论卡
→ 结论卡转为任务卡
→ 任务卡进入创意执行侧重点
→ 生成节点产出结果
→ 结果节点进入资产记忆
→ 成功结果回流主线
→ 失败结果腐殖化为经验
→ 主线重构
```

全程不需要离开当前工作台。

## 7. UI 含义

顶部可以提供轻量焦点模式：

```text
生长 | 创意 | 资产 | 任务 | 复盘 | 历史
```

但它不是页面切换，而是：

- 调整画布对象显示优先级。
- 调整右侧 Inspector 操作入口。
- 调整底部卡片池筛选类型。
- 调整节点连线显示密度。
- 调整 AI 建议动作。

## 8. 设计原则

核心原则：

> **不是两个工作台，而是一张画布里的不同生长状态。**

进一步表述：

> 用户不是在管理聊天记录，也不是在切换多个工作台，而是在同一张智能画布里不断生长、执行、修剪和沉淀。

## 9. 对后续设计的影响

后续 UI、数据结构和开发实现都应遵守以下约束：

1. 不应把创意执行做成独立产品空间。
2. 不应把 TapNow 式节点画布理解为 AI 超级画板的全部。
3. 所有节点、卡片、资产、任务和历史都应属于同一个 Workspace。
4. 焦点模式只改变显示权重和操作入口，不改变对象归属。
5. 创意执行结果必须能回流主线，也能进入资产、历史或腐殖化记录。
6. 思考生长结果必须能转为创意执行输入、任务输入或知识沉淀对象。
