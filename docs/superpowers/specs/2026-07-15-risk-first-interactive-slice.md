# 风险优先交互纵切（Gate 0）

## 决策

先交付一个可在浏览器中完整操作的单用户交互纵切，而不是在未验证交互前一次性引入 PostgreSQL、富文本编辑器、真实模型与多用户同步。该纵切是 Feature 01 的架构验证，不宣称为私有 Alpha 的全部能力。

## 本次范围

用户可以在同一页面完成：

```text
编辑主干文本
→ 在文本框中选择一段文字
→ 创建不可变来源的语义锚点
→ 长出独立分支并写入探索消息
→ 生成结论卡
→ 明确确认后回写主干，产生新修订
→ 将分支设为休眠或腐殖化
→ 查看事件时间线
→ 刷新后恢复本地状态
```

本次以浏览器 `localStorage` 保存工作区；“演示 AI”是确定性的本地建议，不发送文本到网络，也不需要 API 密钥。真实 AI、数据库事务、账号与跨设备同步留在下一 Gate。

## 领域边界

- `Conversation` / `Message`：只追加的讨论记录。
- `TrunkDocument` / `TrunkRevision`：可演进的主干文本；回写总是产生新修订。
- `Anchor`：绑定来源修订、精确引文和 Unicode code point 位置；浏览器 UTF-16 选区在边界转换。
- `Branch`：拥有独立消息和生命周期 `active`、`dormant`、`metabolized`；“已回写”记录在卡片，不作为互斥生命周期。
- `ConclusionCard`：从分支提炼，只有显式“回写主干”才改变主干。
- `WorkspaceEvent`：审计时间线；本次内存实现也以 `commandId` 去重，预留命令收据语义。

## 验收与非目标

验收必须覆盖 Unicode 选区、主干新修订、重复回写不重复追加、分支生命周期和浏览器 Golden Path。非目标包括富文本、React Flow、数据库、真实 OpenAI 调用、认证、多人协同及离线命令队列。
