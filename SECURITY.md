# Security Policy

## Supported versions

This project has not published a stable release. Security fixes target the current `main` branch; older commits and feature branches are not maintained release lines.

## Reporting a vulnerability

Do not disclose a suspected vulnerability in a public issue, discussion, pull request, or commit message.

Use GitHub private vulnerability reporting from the repository's **Security** tab when **Report a vulnerability** is available. If that private channel is unavailable, contact the repository owner, [@ArchitectureWorld](https://github.com/ArchitectureWorld), through a private contact method listed on the owner's GitHub profile and ask for a secure reporting channel before sending exploit details.

Include:

- the affected commit or component;
- impact and realistic attack conditions;
- minimal reproduction steps or a proof of concept;
- any known mitigation or workaround; and
- a private way to coordinate follow-up.

The maintainer will acknowledge receipt, validate scope, coordinate remediation and disclosure, and credit the reporter when requested and appropriate. Public disclosure should wait until a fix or mitigation is available.

Never include real API keys, credentials, personal data, or production data in a report or reproduction.
